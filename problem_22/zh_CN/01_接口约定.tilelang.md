你必须在提交的 Python 代码中提供 `run_kernel` 函数，函数名、参数顺序、类型必须完全一致：

```python
import tilelang
import tilelang.language as T
from tilelang import jit

real_kernel = None

@jit
def per_token_quant_fp8(*args):
    @T.prim_func
    def kernel(*args):
        ...
    return kernel

def run_kernel(
    input,  # Tensor[dtype 见参数说明]
    output_q,  # Tensor[float8_e4m3fn]
    scale,  # Tensor[float32]
    M,  # int64
    N,  # int64
    dtype_code,  # int64
):
    global real_kernel
    if real_kernel is None:
        real_kernel = per_token_quant_fp8(...)
    real_kernel(input, output_q, scale, M, N, dtype_code)
```

### 参数说明

* `input`：输入 tensor，shape `(M, N)`，数据类型由 `dtype_code` 指定
* `output_q`：输出 tensor，shape `(M, N)`，类型为 `float8_e4m3fn`
* `scale`：输出 tensor，shape `(M,)`，类型为 `float32`
* `M`：输入行数
* `N`：输入列数
* `dtype_code`：输入类型编码，`0/1` 分别对应 `bfloat16/float16`

`run_kernel` 内部需要完成以下工作：

* 根据矩阵尺寸计算合适的 launch 配置
* 调用你实现的 TileLang kernel，或使用等价的 GPU Tensor 算子，完成逐行动态量化

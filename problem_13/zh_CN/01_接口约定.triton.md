你必须在提交的 Python 代码中提供 `run_kernel` 函数，函数名、参数顺序、类型必须完全一致：

```python
import triton
import triton.language as tl

@triton.jit
def your_kernel(...):
    ...

def run_kernel(
    a_data,  # Tensor[float8_e4m3fn]
    a_sf,  # Tensor[float32]
    b_data,  # Tensor[float8_e4m3fn]
    b_sf,  # Tensor[float32]
    m_indices,  # Tensor[int32]
    D,  # Tensor[bfloat16]
    M_total,  # int64
    K,  # int64
    N,  # int64
    num_groups,  # int64
):
    ...
```

### 参数说明

* `a_data`：输入 tensor（FP8E4M3），shape $(M_{total}, K)$，行优先连续存储，只读
* `a_sf`：输入 tensor（float32），shape $(M_{total}, \lceil K/128 \rceil)$，per-token 缩放因子
* `b_data`：输入 tensor（FP8E4M3），shape $(num\_groups, N, K)$，每个组的 FP8 权重矩阵
* `b_sf`：输入 tensor（float32），shape $(num\_groups, \lceil N/128 \rceil, \lceil K/128 \rceil)$，per-block 缩放因子
* `m_indices`：输入 tensor（int32），shape $(M_{total},)$，每行的组索引
* `D`：输出 tensor（bfloat16），shape $(M_{total}, N)$，需写入结果
* `M_total`：A 和 D 的第一维大小
* `K`：A 的最后一维及 B 的最后一维大小
* `N`：B 的第二维及 D 的第二维大小
* `num_groups`：分组数量

`run_kernel` 内部需要完成以下工作：

* 根据数据规模计算合适的 grid 和 block 配置
* launch 你实现的 triton kernel，完成 FP8 反量化和分组矩阵乘法

你必须在提交的 CUDA 源码中提供如下 **C 符号**，函数名、参数类型、顺序必须完全一致，并使用 `extern "C"` 防止 name mangling：

```cpp
#include <stdint.h>
#include <cuda_bf16.h>
#include <cuda_fp8.h>

extern "C" void run_kernel(
    const __nv_fp8_e4m3* a_data,
    const float* a_sf,
    const __nv_fp8_e4m3* b_data,
    const float* b_sf,
    const int32_t* m_indices,
    __nv_bfloat16* D,
    int64_t M_total,
    int64_t K,
    int64_t N,
    int64_t num_groups
);
```

### 参数说明

* `a_data`：输入指针（FP8E4M3），shape $(M_{total}, K)$，行优先连续存储，只读
* `a_sf`：输入指针（float32），shape $(M_{total}, \lceil K/128 \rceil)$，per-token 缩放因子，只读
* `b_data`：输入指针（FP8E4M3），shape $(num\_groups, N, K)$，每个组的 FP8 权重矩阵，只读
* `b_sf`：输入指针（float32），shape $(num\_groups, \lceil N/128 \rceil, \lceil K/128 \rceil)$，per-block 缩放因子，只读
* `m_indices`：输入指针（int32），shape $(M_{total},)$，每行的组索引，-1 表示填充行，只读
* `D`：输出指针（bfloat16），shape $(M_{total}, N)$，行优先连续存储，需写入计算结果
* `M_total`：A 和 D 的第一维大小（int64）
* `K`：A 的最后一维及 B 的最后一维大小（int64）
* `N`：B 的第二维及 D 的第二维大小（int64）
* `num_groups`：分组数量（int64）

`run_kernel` 内部需要完成以下工作：

* 根据数据规模计算合适的 grid 和 block 配置
* launch 你实现的 CUDA `__global__` kernel，完成 FP8 反量化和分组矩阵乘法

你必须在提交的 CUDA 源码中提供如下 **C 符号**，函数名、参数类型、顺序必须完全一致，并使用 `extern "C"` 防止 name mangling：

```cpp
#include <stdint.h>

extern "C" void run_kernel(
    const float* x,
    const float* weight,
    const float* bias,
    const float* conv_states,
    float* output,
    int64_t batch_size,
    int64_t dim,
    int64_t seq_len,
    int64_t kernel_size,
    int64_t has_initial_state,
    int64_t silu_activation
);
```

### 参数说明

* `x`：输入指针（float32），shape $(\text{batch_size}, \text{dim}, \text{seq_len})$，只读
* `weight`：输入指针（float32），shape $(\text{dim}, \text{kernel_size})$，只读
* `bias`：输入指针（float32），shape $(\text{dim},)$，只读
* `conv_states`：输入指针（float32），shape $(\text{batch_size}, \text{dim}, \text{kernel_size} - 1)$，只读
* `output`：输出指针（float32），shape $(\text{batch_size}, \text{dim}, \text{seq_len})$，需写入结果
* `batch_size`：batch 大小（int64）
* `dim`：通道数（int64）
* `seq_len`：序列长度（int64）
* `kernel_size`：卷积核宽度（int64）
* `has_initial_state`：是否使用 `conv_states`（0 或 1）
* `silu_activation`：是否对结果做 SiLU 激活（0 或 1）

`run_kernel` 内部需要完成以下工作：

* 根据数据规模计算合适的 grid 和 block 配置
* launch 你实现的 CUDA `__global__` kernel，完成 causal conv1d forward

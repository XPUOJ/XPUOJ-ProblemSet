你必须在提交的 Python 代码中提供 `run_kernel` 函数，函数名、参数顺序必须完全一致：

```python
import torch
import triton
import triton.language as tl

@triton.jit
def your_kernel(...):
    ...

def run_kernel(a_data, a_sf, b_data, b_sf, masked_m, d, num_groups, max_m, n, k):
    ...
```

### 参数说明

* `a_data`：输入 tensor（FP8E4M3），shape $(num\_groups, max\_m, k)$，行优先连续存储，只读
* `a_sf`：输入 tensor（float32），shape $(num\_groups, max\_m, \lceil K/128 \rceil)$，per-token 缩放因子
* `b_data`：输入 tensor（FP8E4M3），shape $(num\_groups, n, k)$，每个组的 FP8 权重矩阵
* `b_sf`：输入 tensor（float32），shape $(num\_groups, \lceil N/128 \rceil, \lceil K/128 \rceil)$，per-block 缩放因子
* `masked_m`：输入 tensor（int32），shape $(num\_groups,)$，每个组的实际处理行数
* `d`：输出 tensor（bfloat16），shape $(num\_groups, max\_m, n)$，需写入结果
* `num_groups`：分组数量
* `max_m`：每个组的最大行数
* `n`：b 的第二维、d 的第三维大小
* `k`：a 和 b 的最后一维大小

`run_kernel` 内部需要完成以下工作：

* 根据数据规模计算合适的 grid 和 block 配置
* launch 你实现的 triton kernel，完成 FP8 反量化和带掩码的分组矩阵乘法

你必须在提交的 Python 代码中提供 `run_kernel` 函数，函数名、参数顺序、类型必须完全一致：

```python
import triton
import triton.language as tl

@triton.jit
def your_kernel(...):
    ...

def run_kernel(
    A,  # Tensor[float8_e4m3fn]
    A_scale,  # Tensor[float32]
    B,  # Tensor[float8_e4m3fn]
    B_scale,  # Tensor[float32]
    output,  # Tensor[bfloat16]
    M,  # int64
    N,  # int64
    K,  # int64
    BLOCK_SIZE,  # int64
):
    ...
```

### 参数说明

* `A`：输入 tensor，shape `(M, K)`，类型为 `float8_e4m3fn`
* `A_scale`：输入 tensor，shape `(M, K_blocks)`，类型为 `float32`
* `B`：输入 tensor，shape `(N, K)`，类型为 `float8_e4m3fn`
* `B_scale`：输入 tensor，shape `(N_blocks, K_blocks)`，类型为 `float32`
* `output`：输出 tensor，shape `(M, N)`，类型为 `bfloat16`
* `M / N / K / BLOCK_SIZE`：与 CUDA 接口含义一致

`run_kernel` 内部需要完成以下工作：

* 正确按 groupwise scaling 规则展开或索引 scale
* 调用你实现的 Triton kernel，或使用等价的 GPU Tensor 算子，完成反量化与 GEMM 计算

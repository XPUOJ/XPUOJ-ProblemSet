## 数据范围

* `a.dtype == torch.bfloat16`
* `b.dtype == torch.bfloat16`
* `output.dtype == torch.bfloat16`
* `pattern_id in {1, 2, 3}`

## 正确性要求

* 必须根据 `pattern_id` 正确实现对应的 contraction
* 必须正确处理 3 种不同输入布局
* 必须将结果写入 `output`

## 提示

* 3 个 pattern 的输出形状都固定为 `(B, M, N)`
* 区别仅在于 `a` 和 `b` 的最后两维语义不同
* `pattern_id = 1` 退化为标准 batch GEMM
* `pattern_id = 2` 和 `pattern_id = 3` 可以看作对一个输入做 layout 交换后的 GETT
* 评测使用 `torch.einsum` 作为参考实现

## 结果判定

评测会使用如下容差进行比较：

```text
rtol = 1e-2
atol = 1e-2
```

## 评测流程

评测程序会先进行预热，再重复执行多轮取平均时间。
注意：为了保证计时准确，**不建议你在 `run_kernel` 内部做 `cudaDeviceSynchronize()` 或显式同步**。

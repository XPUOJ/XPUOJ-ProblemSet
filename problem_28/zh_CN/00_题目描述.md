你需要实现一个 Hopper 风格的 GETT（Generalized Tensor-Tensor Contraction）算子。

GETT 可以理解为一种比普通矩阵乘法更一般的张量收缩运算。给定两个多维张量 `a` 和 `b`，你需要按照指定的维度匹配关系，对某个约简维度做乘加，并生成输出张量 `output`。这类运算通常可以用 Einstein Summation，也就是 `einsum` 表达式来描述。

原始算子支持任意 Einstein Summation 表达式，例如：

```text
einsum('...,...->...', a, b)
```

但为了适配 OJ，本题不要求你实现任意字符串解析，也不开放完全自由的张量表达式，而是固定支持以下 `3` 个常见的 GETT pattern，并通过 `pattern_id` 进行选择：

```text
pattern_id = 1: output = einsum('bmk,bkn->bmn', a, b)
pattern_id = 2: output = einsum('bkm,bkn->bmn', a, b)
pattern_id = 3: output = einsum('bmk,bnk->bmn', a, b)
```

从计算形式上看：

* `pattern_id = 1` 为最常见的 batch GEMM
* `pattern_id = 2` 可以看成左输入做了转置语义变化后的 GETT
* `pattern_id = 3` 可以看成右输入做了转置语义变化后的 GETT


本题评测程序会调用你实现的 `run_kernel` 函数。在函数内部，你需要根据 `pattern_id` 完成对应的张量收缩计算。

如何提交代码详见[评测指南](/d/2)。
